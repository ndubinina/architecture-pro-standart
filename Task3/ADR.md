### <a name="_3bfxc9a45514"></a>**Функциональные требования**
|**№**|**Действующие лица или системы**|**Use Case**|**Описание**|
| :-: | :- | :- | :- |
| UC1 | Новый клиент на сайте, менеджер в системе колл-центра, сотрудник бэк-офиса в АБС | Подача завки на депозит | 1. Новый клиент переходит на сайт в раздел депозитов 2. Клиент выбирает депозит и подает заявку на выдачу через сайт. Заявка сохраняется в системе кол-центра. 3. Менеджер колл-центра звонит клиенту, озвучивает условия депозита и приглашает в банк для идентификации 4. Менеджер передает заявку на депозит в АБС 5. Сотрудник бэк-офиса обрабатывает заявку в АБС |
| UC2 | Клиент в интернет-банке, Сотрудник бэк-офиса в АБС | Выдача депозита клиенту банка | 1. Клиент авторизуется в интернет-банке. 2. Клиент видит персонализированный список доступных депозитов 3. Клиент выбирает депозит и подает заявку на открытие 4. Система отправляет СМС с кодом подтверждения 5. Клиент подтверждает заявку СМС-кодом 6. Заявка сохраняется в АБС 7. Сотрудник бэк-офиса в АБС подтверждает выдачу депозита 8. Клиент получает СМС с подтверждением заявки на депозит|
| UC3 | Сотрудник бэк-офиса кредитов в АБС  | Расчет ставок депозита | 1. Сотрудник бэк-офисов кредитов рассчитывает ставки и загружает их в АБС 2.Ставки публикуются для сайта и для интернет-банка 3. Сотрудники бэк-офиса депозитов может просматривать ставки по депозитам в АБС |
| UC4 | Сотрудник бэк-офиса кредитов в АБС, Сотрудник бэк-офиса депозитов в АБС | Расчет персонализированных ставок депозита | 1. Сотрудник бэк-офисов депозита запрашивает расчет индивидуальной ставки депозита у отдела бэк-офиса кредитов 2. Сотрудник бэк-офиса кредитов рассчитывает персональные ставки и заносит их в АБС 3. Персональные ставки публикуются для интернет-банка |

### <a name="_u8xz25hbrgql"></a>**Нефункциональные требования**
|**№**|**Требование**|
| :-: | :- |
| R   | Надёжность (Reliability)  |
| R1 | Сервисы должны работать 24/7 |
| R2 | Доступность севисов 99,9% |
| ... | ... |
| P   | Производительность (Performance)   |
| P1 | Ответ по операциям интернет-банка не более 1 секунды |
| ... | ... |
| +R  | + Ограничения (Restrictions)       |
| +R1 | Трафик между сайтом и бэкендом необходимо шифровать так как передается чувствительная информация |
| +R2 | Трафик между интернет-банком и бэкендом необходимо шифровать так как передается чувствительная информация |
| +R3 | Используем MS SQL и Oracle для БД | с учетом текущуей экспертизы команды |
| +R4 | Для задачи открытия депозита в интернет-банке делаем микросервисы |
| +R5 | Нужно избежать прямых вызовов API АБС из интернет-банка | АБС не потянет новую нагрузку |
| +R6 | Нужно предусмотреть распределение нагрузки интернет-банка на 2 ЦОД |
| +R7 | Отказ одного ЦОД не должен приводить к полной недоступности интернет-банка|
